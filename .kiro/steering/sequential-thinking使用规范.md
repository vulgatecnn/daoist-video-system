# Sequential Thinking MCP 使用规范

## 概述

Sequential Thinking 是一个强大的 MCP 工具，用于通过结构化的思维过程来分析和解决复杂问题。

## 使用场景

### 适合使用 Sequential Thinking 的情况：

1. **复杂问题分析**
   - 需要分步骤分解的复杂技术问题
   - 多层次的系统设计决策
   - 需要权衡多个因素的架构选择

2. **调试和故障排查**
   - 复杂的 bug 分析
   - 系统性能问题诊断
   - 多组件交互问题的排查

3. **设计和规划**
   - 新功能的设计思路梳理
   - 重构方案的制定
   - 技术选型的决策过程

4. **代码审查和优化**
   - 复杂代码逻辑的分析
   - 性能优化策略的制定
   - 安全性问题的评估

## 使用原则

### 何时使用：
- 问题涉及多个步骤或层面
- 需要考虑多种可能性和权衡
- 分析过程可能需要调整或修正
- 问题的全貌在开始时不够清晰

### 何时不使用：
- 简单的直接问题（如语法错误）
- 已有明确解决方案的常见问题
- 纯粹的信息查询

## 项目中的应用示例

### 1. 视频处理系统优化
当需要优化视频合成性能时，可以使用 Sequential Thinking 来：
- 分析当前性能瓶颈
- 评估不同优化方案
- 制定实施计划

### 2. API 设计决策
在设计新的 API 端点时：
- 分析用户需求
- 评估不同设计模式
- 考虑向后兼容性
- 制定实现策略

### 3. 错误处理机制改进
改进系统错误处理时：
- 分析现有问题
- 设计新的错误处理流程
- 评估对现有代码的影响
- 制定迁移计划

## 最佳实践

1. **明确问题范围**：在开始思考前，清楚地定义要解决的问题
2. **允许思路调整**：在思考过程中可以修正之前的想法
3. **考虑多种方案**：不要局限于第一个想到的解决方案
4. **验证假设**：对关键假设进行验证
5. **记录决策过程**：保留思考过程，便于后续回顾

## 注意事项

- Sequential Thinking 适合复杂问题，简单问题直接解决即可
- 思考过程可能比较长，适合需要深入分析的场景
- 结果会包含完整的思考过程，有助于理解决策依据
- 可以在思考过程中调整方向和深度
